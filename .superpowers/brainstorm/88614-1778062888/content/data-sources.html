Guest Intelligence — data source options

Cost, impact, and tooling for each review source. Pick which to invest in.

A

Google Reviews

You already have: enrich-google-ratings.ts — pulls ratings + review counts via Places API. Currently gets the score but NOT the review text.

Upgrade needed: Switch to Places API v1 reviews field to pull actual review text (5 most recent per request, or use Outscraper for bulk). ~$3-5/retreat for full history via Outscraper.

Coverage: ~180 of 218 retreats have Google listings. 100-500 reviews each.

Pros

  • Broadest coverage — almost every retreat has them
  • Verified visitors (Google checks location history)
  • Already have the API key and script scaffold
  • Structured star ratings for sentiment scoring

Cons

  • Reviews skew short (1-3 sentences typical)
  • Outscraper bulk costs ~$650 for all 218 retreats
  • No sub-category ratings (just overall stars)

Cost: $650 one-time (Outscraper bulk) or $0.017/retreat (Places API, 5 reviews only)
Impact: ★★★★☆ — breadth play, covers every profile

B

TripAdvisor

You have: Nothing built yet.

Needed: Outscraper TripAdvisor scraper ($0.002/review) or Apify TripAdvisor actor (~$5/1000 reviews). Gets full review text + sub-ratings (Service, Rooms, Location, Cleanliness, Value).

Coverage: ~150 retreats have TA listings. 50-300 reviews each, much longer text than Google.

Pros

  • Longest, most detailed review text (paragraphs, not sentences)
  • Sub-ratings map directly to your 15 scoring categories
  • Travelers who review on TA are your exact ICP — serious planners

Cons

  • No existing script — build from scratch
  • TripAdvisor blocks scrapers aggressively
  • Cost ~$300-500 for full corpus via Outscraper

Cost: $300-500 one-time via Outscraper
Impact: ★★★★★ — highest quality text for synthesis

C

Blogger / Media Reviews

You have: discover-forum-intel.ts — searches Google for Reddit/blog/forum mentions. Returns links but doesn't extract content.

Upgrade needed: Add Firecrawl scraping step — for each discovered URL, scrape the full article text. Then use Claude to extract structured insights (what they liked, disliked, specific program names, room descriptions).

Coverage: Top 50 retreats have 10-30 blog reviews each. Long tail has 1-5.

Pros

  • Richest, most narrative content — full stay descriptions
  • Often includes photos, room details, daily schedules
  • You already have the discovery script
  • Firecrawl is already in your toolchain

Cons

  • Unstructured — needs Claude extraction per article ($0.01-0.05 each)
  • Uneven coverage — Ananda has 30 blog reviews, smaller retreats have 0
  • Attribution/copyright considerations for quoting

Cost: ~$50-100 in Firecrawl + Claude API for top 50 retreats
Impact: ★★★★★ — highest editorial value, but labor-intensive

D

Instagram / TikTok

You have: Nothing built.

Needed: Apify Instagram/TikTok scrapers or manual curation. Pull posts tagged at retreat locations or using retreat hashtags. Embed the best ones.

Coverage: Top retreats have 1,000+ tagged posts. Good for visual proof but thin on text insight.

Pros

  • "See it before you book it" — most persuasive for first-timers
  • Social proof at scale (real people, real photos)
  • Native embeds are free and beautiful

Cons

  • Captions are shallow — "amazing experience 🙏" isn't intelligence
  • Instagram embeds slow down pages (Core Web Vitals hit)
  • Content disappears when users delete posts
  • Apify Instagram scraper: ~$10/1000 posts

Cost: ~$50-100 for initial scrape, ongoing curation time
Impact: ★★★☆☆ — visual proof layer, not intelligence

E

YouTube

You already have: discover-youtube-videos.ts + daily-youtube-discovery.sh — finds retreat videos, categorizes official vs independent, stores in Supabase. youtube_videos field exists on every retreat.

Upgrade needed: Pull auto-generated transcripts via YouTube API, then use Claude to extract structured guest insights from the spoken content. A 15-min vlog contains more usable intelligence than 50 Google reviews.

Coverage: Already discovering videos. Top retreats have 20-50 independent vlogs each.

Pros

  • Script already exists and runs daily
  • Video transcripts are the richest review source — 15 min of unfiltered opinion
  • YouTube API transcript access is free (captions endpoint)
  • Embeds work well and don't kill CWV like IG

Cons

  • Transcript extraction needs new code (YouTube captions API)
  • Claude processing per transcript: ~$0.02-0.05 each
  • Auto-captions have errors (proper nouns especially)

Cost: ~$20-50 in Claude API for top 50 retreats (transcripts are free)
Impact: ★★★★★ — richest signal, lowest marginal cost, already half-built

F

Retreat's Own Testimonials

You have: enrich-from-websites.ts — scrapes retreat websites for missing data. Could be extended to pull testimonial pages.

Needed: Add testimonial page discovery + extraction. Most retreat sites have a /testimonials or /reviews page with curated quotes.

Coverage: ~120 retreats have testimonial pages. 10-50 quotes each.

Pros

  • Script scaffold already exists
  • Often includes specific program names and outcomes
  • Free to scrape — no API cost

Cons

  • Curated = biased toward positive
  • Can't verify authenticity
  • Low editorial value alone — only useful combined with unbiased sources

Cost: ~$0 (Firecrawl credits only)
Impact: ★★☆☆☆ — supplementary, not standalone
