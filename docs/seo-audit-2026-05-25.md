# RetreatVault.com — Technical & On-Page SEO Audit
**Date:** 2026-05-25
**Auditor:** Claude (automated)
**Site:** https://www.retreatvault.com
**Stack:** Next.js 14, Supabase, Vercel

---

## Executive Summary

RetreatVault is in strong shape on fundamentals (sitemap coverage, canonicals, redirects, meta tags) but has four issues worth addressing before a link-building or indexing push:

1. **Homepage data leakage** — full retreat JSON objects (price fields, sub_scores, etc.) are SSR-rendered into the HTML. 176 KB page, 8 occurrences of `price_min_per_night` in homepage HTML. Not a crawl-blocker but does inflate page weight.
2. **Retreat detail pages: 0 pre-rendered at build** — `generateStaticParams` returns `[]`, so all 9,400 retreat pages are ISR on-demand. First-hit pages incur Supabase latency + Vercel cold start. Google will eventually cache them, but crawl efficiency suffers.
3. **Schema gap on retreat detail: no `Review` type** — BreadcrumbList, FAQPage, Product, and HealthAndBeautyBusiness are present and well-formed, but there is no `Review` schema despite 1,682 editorial reviews being a core differentiator. Missing `@type: Review` with `author`, `reviewBody`, and `datePublished`.
4. **Guides have zero internal links to individual retreat pages** — `grep` found 0 `href=` to `/retreats/[slug]` in the guides directory. Guides call `getAllRetreats()` and render `<RetreatCard>` components, but card links are the internal linking mechanism. The issue is that guide pages are statically generated from local data files (`GUIDES`, `EDITORIAL_GUIDES`) without programmatic links to specific retreat slugs in prose body copy.

No critical blocking issues. PageSpeed API returned no data (API quota/key required — run manually via https://pagespeed.web.dev/).

## Issue 1: Homepage HTML Data Leakage

**Severity:** Medium
**Type:** Performance / Crawl Efficiency

**Findings:**
- Homepage HTML: **176,307 bytes** (172 KB)
- `price_min_per_night` key appears **8 times** in SSR HTML, confirming full retreat JSON objects are serialized into the React server component payload
- These are the featured retreats in the `HorizontalScroll` component — each object includes all DB fields: `scores`, `sub_scores`, `coordinates`, `gallery_images`, `youtube_videos`, etc.
- The `__NEXT_DATA__` blob is 0 bytes (App Router uses React Flight protocol, not pages-router `__NEXT_DATA__`)

**Impact:**
- Extra ~50-80 KB of JSON per featured retreat set transferred in initial HTML
- Googlebot must parse it all before rendering; bloats Time to First Byte on cold ISR
- Data fields like `price_min: 0` for unpriced listings appear in raw HTML (minor trust issue)

**Fix:**
- Trim retreat objects passed to `HorizontalScroll` to only fields needed for display: `slug`, `name`, `hero_image_url`, `wrd_score`, `score_tier`, `city`, `country`, `price_min_per_night`
- Create a `RetreatCardData` type and use it at the server component boundary before passing to the client scroll component

## Issue 2: Zero Static Pre-rendering of 9,400 Retreat Detail Pages

**Severity:** High
**Type:** Crawl Efficiency / TTFB

**Findings:**
```ts
// src/app/retreats/[slug]/page.tsx
export async function generateStaticParams() {
  // Skip static generation entirely — all pages render on-demand via ISR.
  // getAllRetreats() causes Supabase statement timeouts on Vercel build workers.
  return [];
}
export const dynamicParams = true;
export const revalidate = 86400;
```

- All 9,400 retreat pages are on-demand ISR — **zero pre-built**
- Revalidation is set to 86400s (24 hours) — comment says "refresh every hour" but value is wrong
- Sitemap has **9,900 total URLs** across 11 child sitemaps (455 + ten 1000-URL shards); Google must crawl-trigger all cold renders
- Category pages (city, region, country) also have no `generateStaticParams` — fully on-demand too

**Impact:**
- First Googlebot hit on any uncached page: Supabase query + Vercel SSR latency (est. 500–2000ms TTFB depending on cold start)
- 9,400 cold renders on initial crawl = slow index acquisition
- Comment/code mismatch (`// refresh every hour` with `86400` = 24hr) suggests copy-paste error

**Fix — Option A (preferred):** Pre-render top 500 retreat pages at build
```ts
export async function generateStaticParams() {
  const retreats = await getTopRetreats(500); // order by wrd_score desc, limit 500
  return retreats.map(r => ({ slug: r.slug }));
}
```

**Fix — Option B:** Warm the ISR cache via a post-deploy script that hits the top N retreat URLs. Lower infra cost than full static build.

**Fix — revalidate correction:**
```ts
export const revalidate = 3600; // 1 hour (was incorrectly set to 86400)
```

## Issue 3: Missing `Review` Schema on Detail Pages

**Severity:** Medium
**Type:** On-Page / Structured Data

**Findings:**

Schema blocks present on retreat detail page (`/retreats/1-team-fitness-retreat`):
- `BreadcrumbList` — present, correctly formed
- `FAQPage` with 8 Q&A pairs — present
- `Product` with `AggregateOffer` and `AggregateRating` — present
- `HealthAndBeautyBusiness` with `PostalAddress` and `AggregateRating` — present

**Missing:**
- `@type: Review` — the 1,682 editorial reviews are the site's strongest E-E-A-T signal and should be marked up
- No `author` entity on editorial reviews (analytical chemist reviewer not named in schema)

**Impact:**
- Review-rich results not eligible — Google can't surface star ratings in SERPs without `Review` or `AggregateRating` on a `Review` type
- `AggregateRating` is present on `Product` type but the underlying reviews aren't marked up, weakening E-E-A-T signal

**Fix:** Add `Review` schema for the editorial review on each detail page:
```json
{
  "@context": "https://schema.org",
  "@type": "Review",
  "itemReviewed": {
    "@type": "HealthAndBeautyBusiness",
    "name": "[retreat name]"
  },
  "author": {
    "@type": "Person",
    "name": "RetreatVault Editorial Team"
  },
  "reviewRating": {
    "@type": "Rating",
    "ratingValue": "[wrd_score]",
    "bestRating": "10"
  },
  "datePublished": "[last_updated date]",
  "reviewBody": "[editorial summary text]"
}
```

## Issue 4: Guides Have No Inline Links to Retreat Detail Pages

**Severity:** Medium
**Type:** Internal Linking / PageRank Distribution

**Findings:**
```
grep -r 'href=' src/app/guides/ | grep '/retreats/' | wc -l
→ 0
```

- Guide pages (`/guides/[slug]`) are statically generated from `GUIDES` and `EDITORIAL_GUIDES` data arrays
- Guides do render `<RetreatCard>` components (which link to detail pages via card click), but these are filtered retreat lists — not prose links
- No guide contains inline contextual anchor text like `<a href="/retreats/[slug]">[Retreat Name]</a>` embedded in editorial copy
- 26 guide URLs are in the sitemap (in `sitemap/0.xml`), but their link equity flows to the directory grid, not to specific retreat detail pages

**Impact:**
- High-authority guide pages don't pass PageRank to specific retreat detail pages via anchor text
- Missed opportunity for topical relevance signals (e.g., "Best Yoga Retreats in Bali" guide should link to specific Bali yoga retreat slug pages in body copy)
- `<RetreatCard>` links exist but are JS-rendered card elements — lower weight than `<a>` in prose

**Fix:**
- Add "Editor's Top Pick" section to each guide with 2-3 explicit prose links:
  ```tsx
  <p>Our top pick for this category is <Link href="/retreats/ananda-in-the-himalayas">Ananda in the Himalayas</Link>, which scored 9.4/10 overall.</p>
  ```
- Or add a `featured_retreat_slugs` field to each guide data object and render linked callout boxes

## Issue 5: www Redirect Chain (Minor)

**Severity:** Low
**Type:** Technical / Redirect

**Findings:**
```
http://retreatvault.com
  → 308 → https://retreatvault.com/  (Vercel)
  → 307 → https://www.retreatvault.com/  (Vercel canonical domain redirect)
  → 200
```

- Non-www HTTP hits a **2-hop redirect chain**: 308 (HTTP→HTTPS) + 307 (non-www→www)
- The 307 is a temporary redirect — should be 301 for SEO permanence
- `http://www.retreatvault.com` → 308 → `https://www.retreatvault.com/` → 200 (clean, 1 hop)

**Impact:** Minor crawl budget waste, potential link equity loss through temporary redirect. Vercel manages these — check domain configuration in Vercel dashboard.

**Fix:** Ensure Vercel's non-www → www redirect issues a 308 or 301, not 307.

## Issue 6: OG Image Is a Logo, Not a Preview Image

**Severity:** Low
**Type:** On-Page / Social Sharing

**Findings:**
- Homepage `og:image`: `https://www.retreatvault.com/logo.png`
- Twitter detail page image also falls back to `/logo.png` instead of the retreat hero image
- Retreat detail `og:image` uses the actual hero image (from `retreat.guru`) — correct
- Homepage og:image should be a proper social preview (1200x630 lifestyle image), not the logo

**Fix:** Create a static `og-preview.jpg` (1200x630) for the homepage and global fallback. Logo-only OG images have ~40% lower CTR on social shares.

## Issue 7: `revalidate` Comment/Value Mismatch

**Severity:** Low
**Type:** Code Quality / Bug

**Findings:**
```ts
// src/app/retreats/[slug]/page.tsx
export const revalidate = 86400; // refresh every hour
```

- Comment says "every hour" — value is 24 hours (86400 seconds)
- Same pattern in `city/[city]/page.tsx`, `region/[region]/page.tsx`, `country/[country]/page.tsx`
- Intent is unclear: if 1-hour ISR was desired, retreat pages are stale 24x longer than intended

**Fix:** Decide on ISR strategy and align code + comments:
```ts
export const revalidate = 3600;  // 1 hour
// or
export const revalidate = 86400; // 24 hours
```

## What's Working Well

- **Sitemap:** Proper sitemap index with 11 child sitemaps, 9,900 total URLs, `lastmod` dates present, sitemap referenced in `robots.txt`
- **robots.txt:** Clean — all crawlers allowed, AI bots (GPTBot, ClaudeBot, PerplexityBot) explicitly permitted, only `/api/` disallowed
- **Canonical tags:** Present on all checked pages, correctly formed as absolute URLs
- **Meta titles/descriptions:** Unique, templated, contain score data (e.g., "Rated 8/10 (Exceptional)"), within length limits
- **OG/Twitter tags:** Full Open Graph + Twitter card on all pages
- **HTTPS:** All pages served over HTTPS, HTTP redirects in place
- **Schema breadth:** Retreat detail pages have 4 schema types (Breadcrumb, FAQ, Product, HealthAndBeautyBusiness) — well above average for directory listings
- **ISR architecture:** `dynamicParams = true` correctly set — non-prebuilt pages render on demand and are indexable
- **Guides in sitemap:** 26 guide URLs indexed in sitemap/0.xml
